Angular Routing Introduction

About The Experiment

In this experiment I have tried my hands on main functionality of Angular, which is Single Page Application. Here there is a navigation-bar. When user clicks on different tabs, instead of redirected to other pages, those pages are loaded in the same page through ng-view. So here all components need not to be reloaded. Only those components which are changed, are reloaded.

  • Home
  • About
  • Profile

Code Snippet

sampleApp.config(['$routeProvider',
  function ($routeProvider) {
      $routeProvider.
        when('/home', {
            templateUrl: 'templates/home.html',
            controller: 'homeController'
        }).
        when('/about', {
            templateUrl: 'templates/about.html',
            controller: 'aboutController'
        }).
        when('/profile', {
            templateUrl: 'templates/profile.html',
            controller: 'profileController'
        }).
        otherwise({
            redirectTo: '/home'
        });
  }]);

Description

In this code snippet I have shown the routing controller which routes through different pages. config is inbuilt property of angular. It is used to provide control over routing. When user clicks on some navigation link, the conroller checks it against its list and transferts to the match found. To show the different nav, ng-view is used, which displays dynamically loaded content.
